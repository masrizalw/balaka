## Highlights

Security release. Spring Boot 4.0.4 → 4.0.6 plus Tomcat and PostgreSQL overrides close 30 Dependabot alerts (10 critical, 11 high, 5 moderate, 3 low), spanning the Thymeleaf expression sandbox, Tomcat security-constraint/auth handling, Spring Boot actuator authorization, Spring Security path matching, Spring MVC static-resource handling, and the PostgreSQL SCRAM authentication path. Also includes two tax-rounding corrections aligned with PER-11/PJ/2025 and a fix for an invoice form 500 on validation re-render.

## What's New

### Demo Instances
- `demo.external-db` property lets demo data loaders target an external database instead of the embedded one
- `app.demo-mode` flag emitted for demo instances

## Improvements

- CI actions bumped to their first Node 24 major (`actions/checkout`, `actions/setup-java`, `actions/upload-artifact` → v5) ahead of GitHub's 2026-06-02 Node 20 deprecation
- Deploy `common` role installs `acl` and the `id_ID.UTF-8` locale
- `RecurringSchedulerTest.shouldSkipWeekend` made date-independent (clamps `dayOfMonth` to the entity's 1–28 constraint)

## Bug Fixes

- BUG-020: tax rounding uses `HALF_UP` per PER-11/PJ/2025, preserving the input line
- BUG-019: sub-rupiah rounding residual absorbed into the largest line
- Invoice form no longer returns 500 when re-rendered after a validation error
- User manual: report screenshots pass fiscal-year-2025 date params; sidebar scrolls independently; `.md` cross-references rewritten to `.html`
